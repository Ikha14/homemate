<objective>
## Goal
Toute introspection process (`ps -o … -p …`, ignoré par le ps MSYS) passe par un wrapper portable unique ; les 6 clusters de l'audit migrent dessus sans changer le comportement POSIX.

## Purpose
Windowsisme le plus systémique (risque majeur n°1 de l'audit) : sans lui, identité de lock, détection harness/backend, garde afk et nudge sont tous morts sous Git Bash.

## Output
- `bin/fm-ps-lib.sh` (nouvelle lib) : `fm_ps_field <pid> <comm|args|ppid|pgid>` + `fm_ps_identity <pid>`
- 6 fichiers migrés (audit L36-41 : fm-lock, fm-harness, fm-backend, fm-afk-start, fm-sessionstart-nudge, fm-wake-lib)
</objective>

<context>
## Données empiriques (ticket 05 + sondage session)
- MSYS procfs : `/proc/<pid>/cmdline` (args, NUL-séparé), `/proc/<pid>/exename` (comm), `/proc/<pid>/stat` (ppid champ 4, pgid champ 5, starttime champ 22 — parser APRÈS la parenthèse fermante du comm, qui peut contenir des espaces).
- Process natifs Windows (herdr.exe, Claude Code node) : INVISIBLES du procfs/ps MSYS ; le bash top-level a ppid=1. Fallback comm : `tasklist /FI "PID eq N" /FO CSV /NH`. Détection harness sous Windows = marqueurs d'env (CLAUDECODE=1) en couche 1 — la remontée d'ancêtres s'arrête proprement.
- Identité anti-recyclage (`lstart` indisponible) : starttime procfs + cmdline quand dispo ; sinon dégradation `kill -0` + âge heartbeat (risque résiduel documenté, décision spec).
- POSIX/macOS : passthrough `ps` réel — zéro changement de comportement.
</context>

<acceptance_criteria>
## AC-1: Wrapper correct sous MSYS
Given un process bash vivant (pid MSYS)

Then comm=bash, args=ligne de commande réelle, ppid/pgid numériques cohérents avec /proc ; pid mort → rc≠0 ; pid natif invisible → comm via tasklist ou rc≠0 propre (jamais de sortie poubelle)

## AC-2: Call sites migrés, comportement préservé
Given les 6 fichiers de l'audit
When `grep -n 'ps -o\|ps -p' bin/*.sh` (hors périphérie exclue)
Then zéro occurrence directe restante dans le scope ; `bash -n` vert partout ; sous CLAUDECODE=1 `fm-harness.sh` détecte `claude`

## AC-3: fm_pid_identity portable
Given fm-wake-lib.sh sourcée sous Git Bash
When `fm_pid_identity $$`
Then identité stable non vide (starttime+cmdline), deux appels identiques ; pid mort → rc 1
</acceptance_criteria>
